> [!TIP]
> **Web Development & HMI Testing**: When developing interactive prototypes, always verify that your touch targets are at least 44x44 pixels (Mobile Accessibility Guideline) and ensure your color contrast meets WCAG 2.1 AA standards (minimum 4.5:1 ratio). Testing your interface with actual users using a 'Think Aloud' protocol is essential for identifying usability friction points.

> [!WARNING]
> **Browser Compatibility**: Ensure your implementations are tested across modern Chromium-based browsers (Chrome/Edge). Since many experiments utilize CSS Grid and Flexbox for responsive layouts, legacy browser support may require additional polyfills. Always use relative units (rem/em/%) rather than absolute pixels to maintain interaction consistency across different screen resolutions.

## How to Use

### Viewing Implementation
1. **Navigate** to the specific experiment folder in this laboratory.
2. **Open** the `.html` file directly in a modern web browser (Chrome or Edge).
3. **Interact** with the interface to evaluate responsiveness, feedback, and navigation flow.

**Example:**
```bash
cd "HMI Lab/Experiment-4"
open "index.html" # In Chrome/Edge
```

### Development Setup
**Required Tools:**
- **Text Editor**: Visual Studio Code (Recommended with 'Live Server' extension).
- **Asset Management**: Organize images and stylesheets in local subdirectories.
- **Console Debugging**: Use Browser Developer Tools (`F12`) to evaluate JS event listeners and CSS layout constraints.

### Laboratory Reports
Each experiment includes a comprehensive PDF report covering:
- **Problem Statement**: The HMI challenge or domain analysis being addressed.
- **Design Methodology**: Task analysis, interface styling, and user persona considerations.
- **Implementation**: Web development approach and UI/UX design choices.
- Use these reports as a reference for understanding the theoretical foundations and implementation decisions.

## Learning Path

### Phase 1: Foundations & Analysis
Building the foundation for user-centric design through benchmarking and static layouts.
- **Experiment-1**: Perform Domain Analysis and Rating to understand existing UI standards.
- **Experiment-2**: Develop a Mathematical Application for Kids focusing on cognitive simplicity and engagement.

### Phase 2: Interactive Systems
Implementing dynamic interfaces with complex interaction models.
- **Experiment-3**: Design a Data Entry Form with validation and real-time feedback.
- **Experiment-4**: Implement an ATVM Interface focusing on hardware manipulation and transaction flows.
- **Experiment-5**: Perform an Interface Redesign for a troublesome real-world machine.

### Phase 3: Requirements Engineering
Advanced web architectures and methodological requirement gathering.
- **Experiment-6**: Develop a multi-page E-Commerce Website with consistent navigation.
- **Experiments 7 & 8**: Design complex Menu Layouts and hierarchical navigation schemes.
- **Experiment-9**: Conduct SME Interview Questions to gather precise functional and non-functional requirements.
